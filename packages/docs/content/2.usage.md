# Usage

Run `eks` from any directory inside a project that has a `Makefile`, `package.json`, `deno.json`, or `deno.jsonc`. `eks` walks up the directory tree to find them.

## Single-pick (default)

```sh
eks
```

`eks` discovers every `Makefile` target, `package.json` script, and Deno task from `deno.json` or `deno.jsonc`, opens a fuzzy-searchable picker, and runs the script you select via `dash -c`.

When a discovered Deno config declares `workspace` members, `eks` includes their tasks and renders them as `deno task --cwd <member> <task>`.

## Multi-pick (`--multiple`)

```sh
eks --multiple
```

`eks --multiple` writes the discovered scripts to a temporary batch file and opens it in your [configured editor](/eks/configuration). When you save and exit, the file is parsed and executed using these rules:

- **Uncommented lines** are executed.
- **Comments** (lines starting with `#`) are skipped — comment out anything you do not want to run.
- **Blank lines separate sequential batches** — the next batch only starts after the previous one finishes.
- **Lines within a single batch run concurrently** — group commands that are safe to run in parallel together.

### Example batch file

```sh
# Run lint and typecheck in parallel, then run tests.
deno task lint
deno task typecheck

deno task test
```

This runs `lint` and `typecheck` concurrently, waits for both to finish, then runs `test`.

### Batch failure handling

If any command in a batch fails, `eks` opens the captured stdout/stderr logs in your editor on a best-effort basis. See [Troubleshooting](/eks/troubleshooting) for the log-file naming convention.
