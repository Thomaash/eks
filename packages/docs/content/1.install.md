# Install

`eks` is distributed on JSR as [`@tomina/eks`](https://jsr.io/@tomina/eks) and is installed as a Deno global executable. Deno 2.x is required.

## Install via `deno install`

```sh
deno install \
  --global \
  --allow-env --allow-read --allow-run --allow-sys --allow-write \
  --name eks \
  jsr:@tomina/eks
```

This adds an `eks` binary to your `PATH`. The bare `jsr:@tomina/eks` spec resolves to the package's single export, so no subpath is needed.

## Permissions

`eks` requires the following Deno permission flags:

| Flag            | Reason                                                                             |
| --------------- | ---------------------------------------------------------------------------------- |
| `--allow-env`   | Read `HOME` for path display, and `VISUAL`/`EDITOR` for editor resolution          |
| `--allow-read`  | Read `Makefile`, `package.json`, `deno.json`, and `deno.jsonc` to discover scripts |
| `--allow-run`   | Execute the selected script                                                        |
| `--allow-sys`   | Query terminal size for the picker UI                                              |
| `--allow-write` | Create the temp batch file for `--multiple` mode (unused in single-pick)           |

If a permission is missing, Deno refuses to start `eks` with a `NotCapable` error. Re-install with the missing flag to fix it. See [Troubleshooting](/eks/troubleshooting) for more.
