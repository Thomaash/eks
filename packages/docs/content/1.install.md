# Install via JSR

Install `eks` as a global executable using `deno install`.

## Install eks

```sh
deno install \
  --global \
  --allow-env --allow-read --allow-run --allow-sys --allow-write \
  --name eks \
  jsr:@tomina/eks/bin/eks.ts
```

## Usage

After installing, run from any project directory:

```sh
eks             # fuzzy-pick a single script and run it
eks --multiple  # select multiple scripts to run in sequence
```

## Permissions

The `eks` command requires the following Deno permission flags:

| Flag            | Reason                                             |
| --------------- | -------------------------------------------------- |
| `--allow-env`   | Read `HOME` for path display                       |
| `--allow-read`  | Read `Makefile` and `package.json` to find scripts |
| `--allow-run`   | Execute the selected script                        |
| `--allow-sys`   | Query terminal size for the picker UI              |
| `--allow-write` | Required for `--multiple` mode to create temp file |

The `--allow-write` permission is only used when running with `--multiple` flag.

If a permission is missing, Deno will refuse to start the command with a `NotCapable` error.
Re-install with the missing flag to fix it.
