# Configuration

`eks` needs an editor for two things:

- Editing the batch file in `--multiple` mode.
- Opening failure logs after a failed batch.

The same resolved editor is used for both.

## `--editor <executable>`

Override the editor for a single invocation:

```sh
eks --multiple --editor nvim
```

## Resolution precedence

`eks` picks the editor by trying, in order:

1. `--editor <executable>` flag
2. `$VISUAL` environment variable
3. `$EDITOR` environment variable
4. `nano` (fallback)

## Single-executable only

The editor name is passed directly to the OS as a single executable. Multi-word shell strings like `code --wait` are **not** supported — `eks` does not parse the value as a shell command. Use a plain executable name such as:

- `nvim`
- `vim`
- `nano`
- `code`
- `emacs`

If your preferred workflow needs flags, wrap them in a wrapper script on your `PATH` and pass that script's name to `--editor`.
