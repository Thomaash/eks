# Troubleshooting

## `NotCapable` / missing permission

Deno refuses to start `eks` with a `NotCapable` error when a required permission flag is missing from the install. Re-install with the full set of flags from the [Install](/eks/install) page:

```sh
deno install \
  --global \
  --allow-env --allow-read --allow-run --allow-sys --allow-write \
  --name eks \
  jsr:@tomina/eks
```

## Editor will not launch

If `eks --multiple` or the failure-log viewer cannot start your editor:

- Confirm the resolved editor is on your `PATH` (`which nvim`, `which code`, etc.).
- Pass `--editor <name>` explicitly to override `$VISUAL`/`$EDITOR`.
- Remember: only single executable names work — see [Configuration](/eks/configuration).

If failure-log viewing itself fails, `eks` still completes cleanup and re-throws the original script execution error so it is not lost.

## No scripts found

`eks` walks up from the current directory looking for `Makefile`, `package.json`, `deno.json`, or `deno.jsonc`. If it finds none, it has nothing to list. Make sure at least one exists in the current directory or an ancestor.

## Batch command failures

When a batch in `--multiple` mode fails, `eks` captures each command's output to a temp directory and reopens your editor on the log files. Log files are named:

- `<index>.stdout.log`
- `<index>.stderr.log`

The `<index>` matches the position of the failing command within the batch. Inspect the relevant pair to see what went wrong.
